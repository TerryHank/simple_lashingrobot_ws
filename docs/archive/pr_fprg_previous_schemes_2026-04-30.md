# PR-FPRG 既有方案归档（2026-04-30）

本文档用于冻结本轮会话之前已经探索过的视觉路径，避免后续实验把旧结论混进主链。归档不代表删除代码，含义是：这些方案保留为对照、复盘和后续改进素材，现场主链不回滚。

## 2026-04-30 运行口径更新

- 固定识别位姿扫描仍以 2026-04-22 `PR-FPRG` 拓扑恢复方案作为触发视觉：移动到识别位姿后调用 `MODE_SCAN_ONLY`，生成 `pseudo_slam_points.json` 和 `pseudo_slam_bind_path.json`。
- 执行层局部视觉已从扫描主链拆开：到达每个执行区域后调用 `MODE_EXECUTION_REFINE`，走平面分割 + Hough，仅负责局部修正和执行框内点输出。
- 本文下面的“当前主链”只描述扫描建图 / 视觉研究主链，不表示执行层也使用 `PR-FPRG`。

## 当前扫描主链

- 方案 1：行/列峰值 PR-FPRG。手动工作区透视展开后，在 rectified 工作区内固定对行、列 profile 做峰值识别，生成正交网格，再投回原图输出绑扎点。
- 底层图像优先使用组合响应：深度暗线响应 + 红外暗线响应。用户明确认可该响应图对钢筋更清楚，后续报告和调试应默认展示组合响应，同时保留深度响应作为派生模态。
- 方案 1 的目标是稳定出规划绑扎点，不再主动加入角度搜索、局部曲线收束或旧 pre_img 门控。

## 废案与保留对照

- 方案 2 废案：红外最终 rho 微调及其相关分支。该路径曾用于尝试修正局部偏移，但效果和鲁棒性不满足当前尺度要求，不再作为主链候选。
- 方案 3/4/5/6：曲线跟踪、DP 深度曲线、DP ridge 曲线、红外辅助曲线等方案只作为后续收束研究保留。当前观察是曲线容易被地板缝、TCP 侵入或局部强响应牵引，直接替换主链会破坏用户认可的方案 1 效果。
- FFT 版行/列峰值：保留为周期估计对照。它适合检查 profile 周期谱和峰值驼峰，但不是方案 3/4/5/6 的默认主链。
- 旧 RANSAC + Hough + pre_img：已另行归档到 `docs/archive/legacy_ransac_hough_pointai/`，不恢复为扫描主链门控；执行层的平面分割 + Hough 使用独立运行模块承接，不直接导入旧 `pre_img()`。

## 已知限制

- 梁筋 edge-band mask 只是宽强响应边缘带检测，不是实例分割。它主要依赖工作区边缘附近的异常宽、异常强结构，可能漏掉内部梁筋、弱梁筋，或把 TCP 侵入造成的横向强响应误看成结构带。
- 间距筛选只能作为近重复线清理，不能假设现场钢筋间距完全规则。小于阈值的真实密集钢筋可能被误删，因此后续要优先用响应证据、实例图和物理尺度约束，而不是硬间距过滤。
- 「钢筋摆在地上本身是斜的」已经记录为后续问题：它不是相机画面歪斜导致的透视问题，未来需要在实例分割或 3D/BEV 层面处理真实弯曲、倾斜和穿梁筋情况。

## 下一步实验方向

- 组合响应和深度响应继续往下衍射：从响应图派生多尺度 Hessian/Frangi-like 脊线、骨架、端点/交叉点图、梁筋候选宽带、worldCoord 高度/深度梯度等模态。
- 新实验以「钢筋实例图」为目标，而不是只找理论直线交点：先把钢筋中心线、梁筋候选和普通钢筋候选分开，再在梁筋 ±13 cm 区域之外生成绑扎点。
- 所有新路线先以独立报告和工具呈现，确认效果后再讨论是否进入主链。
