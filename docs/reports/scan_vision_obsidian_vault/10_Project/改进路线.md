# 改进路线

## 优先级 1：先阻止坏扫描写账本

目标：避免 816 个原始点、540 个规划成员这类结果进入执行层。

建议加入质量门：

- 原始点数超出合理范围时拒绝写入。
- 规划成员数超出合理范围时拒绝写入。
- `规划成员数 / 唯一 cell 数` 大于阈值时拒绝写入。
- 单个 cell 候选数超过 1 或 2 时记录严重告警。
- 周期估计低于物理最小间距时拒绝写入。

对当前现场，真实点约 238 个，可以先设：

```text
软范围：180~300
硬范围：120~360
```

后续再根据现场真实规格调整。

## 优先级 2：修周期估计

当前强证据显示周期可能选到半周期。

建议：

1. 把 `min_period` / `max_period` 从像素常量改成真实钢筋间距范围。
2. 用 `resolution_mm_per_px` 把 120~180 mm 转成 px。
3. 加入谐波判别：如果 `p` 和 `2p` 都有响应，且 `2p` 符合物理间距，优先选 `2p`。
4. 用预期点数约束线数，避免线数乘法爆炸。

## 优先级 3：每个 cell 只留一个点

当前规划成员有 540 个，但唯一 cell 只有 194 个，说明重复候选是大问题。

建议：

- 在视觉层或流程层构建 `(row, col)` cell。
- 每个 cell 内按局部响应分数、深度稳定性、距离 cell 中心的偏差排序。
- 只保留一个最高置信候选。
- 输出被抑制点数量，用于调试。

## 优先级 4：加局部真实交点验证

候选交点附近开小 ROI，计算：

- 两个方向是否都有线状 ridge 支撑。
- 两条线是否在候选点附近相交。
- depth 响应是否有局部峰值或脊线。
- IR 响应是否支持，但不直接覆盖最终位置。
- ROI 内有效深度比例是否足够。

低置信候选拒绝输出。

## 优先级 5：从生成式网格升级到检测式网格

长期方案：

1. 在 rectified depth response 上做线状结构增强。
2. 用 Hough / LSD / Steger 检测真实线段。
3. 对两组钢筋线做鲁棒拟合。
4. 只对有线段支撑的线求交。
5. 允许低自由度偏斜或缓慢漂移。

这样能保留 PR-FPRG 的全局几何优势，同时避免「无条件铺满」。

## 优先级 6：引入轻量模型做候选验证

如果传统方法仍受现场光照、反光、遮挡影响，可以训练轻量关键点检测模型：

- 输入：IR / RGB / depth response 的组合图。
- 输出：交点关键点或小框。
- 用途：不是完全替代几何，而是验证和排序候选点。

参考路线：YOLO-FAS、YOLOv8 keypoint、轻量 segmentation。
