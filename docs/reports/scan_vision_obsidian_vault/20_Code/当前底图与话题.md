# 当前底图与话题

## 扫描层当前利用的底图

当前扫描层算法主底图是 **深度图**。

更精确地说，`MODE_SCAN_ONLY` 下的 `manual workspace S2 / PR-FPRG` 使用：

- `self.depth_v`：当前深度图，转为 `np.float32`。
- `raw_world_coord` / `image_raw_world`：用于从输出像素附近取相机三维坐标。
- `manual_workspace_quad.json`：工作区四边形与四角世界坐标。

算法不是直接在 RGB 或 IR 底图上识别绑扎点。

## 当前处理底图

当前主响应图来自 rectified depth：

```python
raw_depth = self.depth_v.astype(np.float32)
rectified_depth = cv2.warpPerspective(raw_depth, forward_h, rectified_size)
background_depth = cv2.GaussianBlur(filled_depth, sigmaX=11.0, sigmaY=11.0)
depth_response_variants = [
    background_depth - filled_depth,
    filled_depth - background_depth,
]
```

系统会分别评分两个 depth-only 响应变体，选择纵横周期估计总分更高的那个。

## IR 图像的角色

IR 图像目前不是扫描主链底图。它主要用于：

- 前端显示。
- 历史或离线报告中的视觉对照。
- 已归档的红外 rho 微调实验。

按照当前 `CHANGELOG.md` 口径，扫描 S2 主链已经回到 2026-04-22 的 depth-only 版本，不再使用 depth+IR 组合响应、红外最终贴线或 line-family 实验链路。
