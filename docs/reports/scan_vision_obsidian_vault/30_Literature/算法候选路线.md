# 算法候选路线

## 路线 A：保守修复当前 PR-FPRG

适合短期现场止血。

改动：

- 物理间距约束周期。
- 谐波检测。
- 每个 cell 只保留一个候选。
- 加质量门。

优点：

- 改动小。
- 不需要训练数据。
- 最快降低误检数量。

风险：

- 如果钢筋线局部弯曲、遮挡或梁筋干扰严重，仅修周期仍不够。

## 路线 B：PR-FPRG + 局部 ridge 验证

适合中期提升精度。

改动：

- 保留 PR-FPRG 生成候选区域。
- 在候选 ROI 中做 ridge / line fitting。
- 用局部拟合交点替换全局网格交点。

优点：

- 兼顾全局拓扑和局部真实证据。
- 与当前架构兼容。

风险：

- 需要设计稳定的局部评分和阈值。
- 需要现场标注少量样例做调参验证。

## 路线 C：线段检测 / Hough 多段拟合主链

适合把扫描层改成检测式。

改动：

- 从 depth response / IR response 中提取真实钢筋线段。
- 聚类成两组线族。
- 求交点并做局部 3D 取点。

优点：

- 文献中已有钢筋绑扎机器人成功案例。
- 对假周期更稳。

风险：

- 对图像质量和阈值敏感。
- 需要处理断裂线段、遮挡和梁筋。

## 路线 D：轻量深度学习验证器

适合长期产品化。

改动：

- 采集现场数据。
- 标注交点、已绑扎状态、梁筋干扰。
- 训练轻量 YOLO / keypoint 模型。
- 与几何算法融合。

优点：

- 对复杂纹理和光照适应性更强。
- 能同时识别交点状态。

风险：

- 需要数据集和训练流程。
- 需要部署算力和模型版本管理。

## 推荐组合

短期：

```text
路线 A + 输出质量门
```

中期：

```text
路线 B
```

长期：

```text
路线 B + 路线 D
```
